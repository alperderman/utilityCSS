utilityCSS

utilityCSS is a purely functional stylesheet consist of only the most essential styles required for a reliable starting point for your design.

What does utilityCSS include:

  • a slightly modified version of normalize.css by necolas
  • an impeccable grid system inspired by Foundation
  • aspect ratio classes
  • positioning classes (absolute, relative, fixed...) and including text and background positions
  • visibility classes
  • basic print styles
  • lastly media queries (medium, large and print) that can be used for all of the features mentioned above

utilityCSS only gives you the basic tools and it tries to be as "unopinionated" as possible, so it DOESN'T include any design specific styles, such as:

  • text and background colors (except for the print layout)
  • sizing classes
  • display classes (except display none included in visibility classes)
  • margin, padding, border classes
  • box and text shadows
  • any kind of pre-made components (such as cards, modals, tooltips etc..)

Grid System

Grid system in utilityCSS is over-engineered to be as consistent as possible for all kind of situations and problems (and of course CSS quirks) regarding to positioning that you might stumble upon while designing the elements of your website. Classes are diluted and always serves a meaningful purpose that tackles each and every single edge-case.

As mentioned before the inspiration for this grid system comes from Foundation, and what I mean by inspiration is; only some of the class names, seperation of rows (grid-x) and columns (grid-y) and media query sizes. Other than that, all of the CSS properties are completely different under the hood and codes between utilityCSS and Foundation are obviously NOT INTERCHANGABLE even though they might look similar.

If you are new to CSS in general, most if not all grid systems for the web use 12 divisions. The reason for that is, 12 is the smallest number that can be divided into wide range of numbers (2, 3, 4 and 6) which grants the ability to shape our design to our liking. The power of 12 can be even seen in our daily lives, for example it is the same reason why we have 12/24 hour clock cycles and 12 months.

As you might noticed, there are also -auto and -fill classes. -auto sets the size dynamically based on the content in the cell and -fill as the name suggests, fills the remaining row or column (depending on if it is grid-x or grid-y). Lastly, a cell without any sizing classes will act like as if they have -auto class.

grid-x and grid-y classes by default, have flex-wrap property set to wrap. Additionally, both grids and cells have max-width and max-height of 100% and overflow is set to auto, so that they can accommodate overflowing children without any data loss.

There are few things to consider, grid-y which is flex column, needs a height property in order to work, that is because, cells are defined with percentage values (except -auto and -fill as they are special). And in CSS, percentage values are based on the size of the parent element. So if you don't set a height to the grid-y element, cells cannot get a reference point for percentage, In the example, we used -frame-y class to set the grid height to viewport, which most of the time thats what it is needed, but might not always be applicable depending on what you are actually designing. For grid-x element, you don't need to set a width because, by default flexbox layout behaves like a block element which automatically takes up all the width space of the parent.

Grid and Cell Property Classes

There are classes that can be attached to grid and cell elements to set certain properties, these are mostly properties unique to flexbox layout in CSS. One difference is however, utilityCSS streamlined the justify-content, align-items and align-self properties with -top, -left, -center... classes, so that it can be understood easier at first glance.

It is not included in the list below, but -debug property class can be attached to any element to see their borders just like in the examples. Borders are color coded; blue for grids, cyan for cells and red for everything else.

Grid Property Classes

Class Description
-full Sets the grid height to 100%.
-frame-x Sets the grid width to viewport.
-frame-y Sets the grid height to viewport.
-frame Sets the grid size to viewport.
-reverse Reverses the cell order.
-overflow Removes max-size limits and forces overflowing content to be visible (You may also need to add -overflow to cells, since they also have max-size limit).
-wrap Automatically divide cells into multiple rows or columns when they exceed the grid size (Default).
-nowrap Disables wrapping so cells are divided into a single row or column.
-wrap-reverse Wrap the rows or columns in reverse order.
-left Horizontally aligns the cells to left (Default).
-center Horizontally aligns the cells to center.
-right Horizontally aligns the cells to right.
-between Equally divides the row or column with spaces in between the cells.
-around Equally divides the row or column with spaces in between and around the cells.
-top Vertically aligns the cells to top.
-middle Vertically aligns the cells to center.
-bottom Vertically aligns the cells to bottom.
-stretch Stretches out the cells to opposite direction of grid.
-baseline Positions the cells to the text base line, where the cells with different font-sizes, will be centered accordingly (hard to explain even harder to visualize, mostly never used but it exists).

Cell Property Classes

Class Description
-[SIZE 1 to 12] Sets the cell to a specific size.
-auto Sets the size of the cell based on content inside.
-fill Fill the remaining space in row (grid-x) or column (grid-y).
-full Sets the height of the cell to 100%.
-frame-x Sets the width of the cell to viewport.
-frame-y Sets the height of the cell to viewport.
-frame Sets the size of the cell to viewport.
-overflow Removes the size limitations and forces the overflowing content inside the cell to be visible.
-push-[SIZE 1 to 12] Adds margin to the cell in grid start direction (left for grid-x, top for grid-y). Reverses if the grid has -reverse class.
-pull-[SIZE 1 to 12] Adds margin to the cell in grid end direction (right for grid-x, bottom for grid-y). Reverses if the grid has -reverse class.
-left Horizontally aligns the cell to left.
-center Horizontally aligns the cell to center.
-right Horizontally aligns the cell to right.
-top Vertically aligns the cell to top.
-middle Vertically aligns the cell to center.
-bottom Vertically aligns the cell to bottom.
-stretch Stretches out the cell in opposite direction of grid. It is default option and if it is used with other positioning classes -stretch will cancel them out depending on if it is grid-x or grid-y.
-baseline Positions the cell to text base line, cancels out other positioning classes.

Aspect Ratio

Aspect ratio classes are defined to set a certain ratio of width and height to an element. There is already an aspect-ratio property in CSS but it only has been added in 2021, which that means if you want to support older browsers, you have to use a different approach.

utilityCSS, utilizes the latter approach in these classes since it correlates better with its aim of "universality".

Visibility

Visibility classes in utilityCSS, enables us to control whether the element is shown or hidden. These classes are usually coupled with media queries.

Class Style
-invisible visibility: invisible;
-visible visibility: visible;
-hide display: none;
-hide_[MEDIA QUERY]-only Adds display: none; only to chosen screen size.
-show_[MEDIA QUERY] Adds display: none; to any size below chosen screen size.
-show_[MEDIA QUERY]-only Adds display: none; to any size except the chosen screen size.

Print Styles

utilityCSS comes with a set of styles for print layout. Using these classes has no effect on normal layout, they only affect when printing is initiated.

Class Description
-print-bw Optimize the element for black and white printing.
-break-before Adds a page break before the element.
-break-after Adds a page break after the element.
-break-avoid Avoids breaking the page inside the element.

Media Queries

Media queries are a way of defining styles based on certain browser conditions, mostly used for defining unique styles for different screen sizes so that the layout can adapt to devices with small screens such as mobile phones or with bigger screens like a television. In web development, we use the term "responsiveness" to define this capability.

utilityCSS comes with a set of media queries for each property class mentioned above, so that different classes can be activated for different screen sizes.

Use suffix _medium for any property class to be activated for only the devices with screen width above 40em (640px) or _large for screen width above 64em (1024px).

Additionally, there is _print suffix for only activating property classes for print layout.

Media Query Description
[PROPERTY CLASS]_medium For screen sizes above 40em (640px).
[PROPERTY CLASS]_large For screen sizes above 64em (1024px).
[PROPERTY CLASS]_print For print layout only.

Example is given below, minimize your browser and resize it to activate different classes.

cell -8 -6_medium -4_large
This cell will only shown on large and above screens
cell -6 -4_medium -2_large -right_large

                        This cell will only shown on large and above screens
                    </div>
